## 设计方向

当前前端设计原则：

- 视觉尽量克制，减少“门户站”式堆叠
- 以阅读与检索效率优先
- 首页只承担入口、搜索和目录作用
- 文档页强调来源、分类与连续阅读
- 兼顾桌面端与移动端

目前视觉基调已从早期的“深浅色切换 + 卡片导航”改为更稳定的浅色阅读界面。

## 功能概述

文档页右侧边栏会自动显示两类链接：

### 1. 准则引用（准则引用）
- 自动提取正文中的 `《企业会计准则第X号——XXX》` 格式引用
- 支持多种破折号格式：`——`、`—`、`－`、`-`
- 自动去重（同一准则多次引用只显示一次）
- 优先显示"现行有效"版本的准则
- 不修改原始 Markdown 内容

### 2. 解释与应用指南
- 自动查找当前准则对应的准则解释文档
- 自动查找当前准则对应的应用指南文档
- 文件命名规范：`XX-准则解释第X号.md`、`XX-XXX应用指南.md`

## 发布新准则后的更新流程

### 步骤1：添加新准则文档

在 `docs/01-企业会计准则/` 目录下添加新准则 Markdown 文件：

```markdown
---
title: "企业会计准则第XX号——XXX"
short_title: "XXX"
date: 2024-01-01
category: "企业会计准则"
number: XX  # 重要：必须填写准则号
status: "现行有效"
summary: "准则摘要"
tags: ["标签"]
origin_url: "https://..."
---
# 企业会计准则第XX号——XXX

正文内容...
```

**注意**：
- `number` 字段必须填写，用于匹配准则引用
- 多个版本（如2006版、2017版）需要分别添加，并设置不同的 `status`

### 步骤2：添加对应的解释和应用指南（可选）

在 `docs/02-企业会计准则解释/` 目录下添加解释文档：
- 文件名格式：`XX-准则解释第X号.md`

在 `docs/03-企业会计准则指南/` 目录下添加应用指南：
- 文件名格式：`XX-XXX应用指南.md`

### 步骤3：构建并部署

```bash
cd site
bun run build
# GitHub Actions 会自动部署
```

## 关键配置项

### 正则表达式

当前使用的正则表达式：
```javascript
/《企业会计准则([^》]+)》/g
```

匹配示例：
- `《企业会计准则——基本准则》` → 提取 "——基本准则"
- `《企业会计准则第22号——金融工具确认和计量》` → 提取 "第22号——金融工具确认和计量"

### 匹配逻辑

1. 从正则匹配结果中提取准则号：`第(\d+)号`
2. 在 `01-企业会计准则/` 目录下查找对应文档
3. 优先选择 `status: "现行有效"` 的版本
4. 排除当前文档自身（避免自引用）

### 文档命名规范

| 类型 | 目录 | 命名格式 |
|------|------|----------|
| 企业会计准则 | `01-企业会计准则/` | `XX-XXX.md` |
| 准则解释 | `02-企业会计准则解释/` | `XX-准则解释第X号.md` |
| 应用指南 | `03-企业会计准则指南/` | `XX-XXX应用指南.md` |

## 故障排查

### 问题：准则引用不显示

1. 检查文档 frontmatter 是否包含 `number` 字段
2. 检查正则表达式是否匹配（参考正则测试工具）
3. 检查目标文档是否存在

### 问题：解释与应用指南不显示

1. 检查文件名是否包含准则号（如 `-22-` 或 `-第22号`）
2. 检查文件是否在正确目录

### 问题：number 字段不生效

在 Astro 中，`doc.data.number` 可能为 `undefined`，代码已兼容处理：
```javascript
const currentNumber = doc.data.number ?? doc.rendered?.metadata?.frontmatter?.number;
```
