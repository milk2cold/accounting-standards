# 03-内容整理 - 模块说明

## 模块目标

维护 `docs/` 作为内容主库，保证准则原文完整、来源可追溯、站点可直接消费。

当前内容整理的核心原则：

1. `docs/` 是主数据源
2. 优先采用官方或高可信来源
3. Markdown 文件应具备稳定 frontmatter
4. 网站内容由同步脚本从 `docs/` 自动导入

```text
docs/
├── 01-企业会计准则
├── 02-企业会计准则解释
├── 03-企业会计准则指南
├── 08-每月更新
└── 09-推荐资源
```

站点不会直接手工维护正文，而是读取并同步这里的 Markdown。

## 当前内容标准

### 来源

优先顺序：

1. 财政部会计司
2. 中国会计准则委员会
3. 德勤 CAS Plus

### 文件要求

- 必须有完整正文
- `origin_url` 应尽量指向官方或高可信来源
- 标题、编号、版本信息应可从文件名和 frontmatter 识别
- 历史版本应明确区分年份

### Frontmatter 最低要求

```yaml
---
title: "企业会计准则第1号——存货"
category: "企业会计准则"
---
```

建议逐步补充：

```yaml
short_title: "存货"
date: 2006-02-15
number: 1
status: "现行有效"
summary: "规范存货的确认、计量和披露。"
origin_url: "https://..."
```

## 与网站的关系

`site/scripts/sync-docs.ts` 会在构建前自动：

- 复制指定目录到 `site/src/content/docs/`
- 忽略 `json`、`pdf`、`docx`、`archive` 等不需要入站的文件
- 为缺少 frontmatter 的 Markdown 自动补最小元数据
- 清理不兼容的 `layout:` 字段

因此，日常内容维护应优先修改 `docs/`，而不是直接改 `site/src/content/docs/`。
